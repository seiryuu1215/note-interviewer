# 収益化設計書

## ビジネスモデル

**SaaS型サブスクリプション** — AIインタビューによる記事生成サービス

### ターゲット
- noteで発信したいが文章を書くのが苦手な人
- 定期的にコンテンツを出したいがネタ出し・構成に時間がかかる人
- インタビュー記事を量産したいメディア運営者

### 競合優位性
- 「タイトルだけ」で始められる圧倒的なハードルの低さ
- 対話型なので自分の言葉が記事に反映される（AIっぽくない）
- プロフィール蓄積で使うほど精度が上がる

## 料金プラン

| プラン | 月額 | 記事数 | 主な機能 |
|---|---|---|---|
| **Free** | ¥0 | 3記事/月 | 基本インタビュー + 記事生成 |
| **Pro** | ¥980 | 30記事/月 | プロフィール蓄積 + テンプレート + 過去記事閲覧 |
| **Business** | ¥2,980 | 無制限 | チーム機能 + API連携 + note直接投稿 + 優先サポート |

### 価格設定の根拠
- Anthropic APIコスト: 1記事あたり約¥10〜30（インタビュー + 記事生成）
- Free枠: 月3記事 = 月¥30〜90のコスト（獲得コストとして許容）
- Pro: 月30記事フル利用で¥300〜900のAPI費。¥980で粗利60%以上
- Business: 無制限だが実際の平均利用は50〜100記事。十分な利益率

## 実装フェーズ

### Phase 1: MVP（完了）
```
現状の実装:
- クライアントサイドのみ（localStorage）
- 無料枠: 月3記事（クライアントサイドカウント）
- ユーザー認証なし
- APIキーは環境変数で固定
```

### Phase 2: 認証 + サーバーサイド管理
```
技術: NextAuth.js + Google OAuth + Supabase

テーブル設計:
- users: id, email, name, avatar_url, plan, created_at
- sessions: id, user_id, title, messages(jsonb), status, created_at
- articles: id, session_id, user_id, title, content, created_at
- usage: user_id, month, interview_count, article_count

実装タスク:
1. Supabase プロジェクト作成 + テーブル定義
2. NextAuth.js + Google OAuth 設定
3. localStorage → Supabase 移行
4. サーバーサイド使用量チェック（API route内）
5. マイページ（生成履歴・プロフィール管理）
```

### Phase 3: Stripe 決済
```
技術: Stripe Checkout + Webhooks

実装タスク:
1. Stripe プロダクト・価格設定（Free/Pro/Business）
2. Checkout セッション作成 API
3. Webhook でサブスク状態を同期
4. プランに応じた使用量制限の適用
5. プランアップグレード・ダウングレードUI
6. 請求ポータルへのリンク
```

### Phase 4: プラットフォーム拡張
```
- note API連携（OAuth + 記事投稿）
- テンプレート機能（文体・構成パターンの保存）
- チーム招待（Business プラン）
- APIキーの代理管理（ユーザーの手間をゼロに）
```

## マーケティング

### ローンチ戦略
1. **Zenn記事**: 技術的なアーキテクチャ解説（エンジニア向け）
2. **note記事**: 「このアプリで書いた記事」をnoteに投稿（実証）

## リスク

| リスク | 影響 | 対策 |
|---|---|---|
| API費用の急騰 | 利益率低下 | キャッシュ活用、モデル選択の最適化 |
| note API制限 | 直接投稿不可 | Markdownコピー機能で代替 |
| 競合参入 | シェア低下 | プロフィール蓄積による差別化 |
| AI規制強化 | サービス制限 | 透明性確保、利用規約整備 |
